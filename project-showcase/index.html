Why Escrow Matters

Without Escrow

  • Client Risk: Pays upfront, student disappears.
  • Student Risk: Works for weeks, client refuses to pay.
  • Result: Zero trust, high friction.

With Escrow

  • Funds Locked: Money is secured before work begins.
  • Trust Increased: Student is guaranteed payment upon approval.
  • Disputes Managed: Admin resolves deadlocks.

Under the Hood: Technical Implementation

These are the actual algorithms powering the recommendation system — not simulated or assumed.

K-Means Clustering (ML, K=4)

An unsupervised ML algorithm groups all students into 4 tiers based on 6 platform metrics. The active tier (highlighted) changes live as you adjust the verification dropdown above.

Beginner New & learning, lower scores
Rising Talent Growing track record, +5% bonus
Professional Consistent delivery, +10% bonus
College Elite Top performers, +15% bonus

6 Features Extracted per Student

Completion Rate On-Time Delivery Total Earnings Projects Completed Avg Client Rating Reputation Score

Jaccard Similarity Index

The Skills Match score is not a simple percentage. The backend computes the exact Jaccard Index — the number of matching skills divided by the total unique skills across both the student and the project.

Score = |A ∩ B| / |A ∪ B|

BullMQ Async Processing

Calculating scores for thousands of student-project pairs is heavy work. The system offloads this to background workers via BullMQ + Redis queues — keeping the API fast and responsive while scores are computed asynchronously.
